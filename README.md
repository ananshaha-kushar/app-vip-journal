# app-vip-journal

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的存储与展示空间。

## 项目简介

app-vip-journal 旨在以简单、直接的方式托管独立的 HTML 文件。每个页面均为自包含的文档，可独立访问，无需依赖外部资源或后端服务。适用于快速分享、存档或演示用途。

## 目录说明

```
.
├── index.html          # 根页面，可自定义为导航或说明页
├── pages/              # 存放归档的 HTML 页面
│   ├── example1.html   # 示例页面
│   └── example2.html   # 示例页面
└── assets/             # 可选的静态资源（CSS、JS、图片等）
    └── style.css
```

- **根目录**：包含首页 `index.html`，可用于列出所有归档页面或提供使用说明。
- **pages/**：主要归档目录，每个 `.html` 文件代表一个独立页面。
- **assets/**：存放页面引用的资源文件，建议在此统一管理样式与脚本。

## 页面归档说明

- 每个 HTML 页面应保持独立，内部引用资源（如 CSS、JS、图片）请使用相对路径，并确保路径正确。
- 页面标题、样式、脚本均可在文件内定义，无需额外配置。
- 若需新增页面，直接在 `pages/` 目录下创建新的 `.html` 文件，并在 `index.html` 中添加链接即可。

## 维护说明

- 本仓库采用静态页面托管方式，无需构建或编译步骤。
- 提交新页面或修改现有页面后，仓库会即时更新。
- 请勿在页面中包含敏感信息或违反服务条款的内容。
- 建议定期整理 `pages/` 目录，删除不再使用的页面以保持整洁。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意页面。
3. 也可将本仓库部署到支持静态页面的托管平台（如 GitHub Pages），实现线上访问。
